Scale for project CPP Module 00

You should evaluate 1 student in this team

Introduction

Please follow the following rules:

- Stay polite, courteous, respectful and during the process
evaluation. The well-being of the community rests On that.

- Identify with the evaluated person or group evaluated
dysfunctions of his work. Take time to discuss it and
discuss problems identified.

- You have to take into account that there may be slight differences
interpretation between the project's instructions, its scope and its
functionalities. Keep an open mind and note in the most
honestly possible. Pedagogy is not valid only if the peer-evaluation
is made seriously.

Guidelines

- Only note what is contained in the student's cloned Git deposit
or group.

- Check that the Git deposit belongs to the student or group,
that the project is indeed the one expected, and that "git clone" is used
in a file empty.

- Check carefully that no aliases has been used to deceive you
and Make sure you evaluate the official rendering.

- In order to avoid any surprises, check with the student or group
potential scripts used to facilitate evaluation (e.g.
test or automation scripts).

- If you haven't done the project you're going to evaluate, you have to read
the whole subject before to start the evaluation.

- Use the available flags to report an empty rendering, a programme
does not work, a standard error, cheating... In these
situations, the assessment is completed and the rating is 0, or -42 in case of
cheat. However, with the exception of cases of cheating, you are encouraged
to continue the discussion on the work rendered, even if the latter is
incomplete. This is in order to identify the errors to be not to reproduce in the
Future.

- If the subject requires a configuration file, you should never
must amend it. If you want to edit a file, take the time
explain why the person being evaluated and make sure you have
its agreement.

- You should also check for no memory leaks. Any memory
allocated on the heap must be released cleanly before the end of the execution
of the programme.
You have the right to use any tool available on the machine such as
leaks, valgrind or e-fence. In case of memory leaks, tick the flag
appropriate.

Prerequisites

The code must compile with c' and the flags -Wall -Wextra -Werror
As a reminder, this project must follow standard C-98. Therefore,
functions C-11 (or other standard) and the containers are NOT
expected.

Do not record the exercise if you find:

  • A function implemented in a header file (except for the template functions).
  • A Makefile compiles without the requested flags and/or with other What a thing c.

Select the flag "Forbidden function" if
you meet:

  • The use of a "C" function (a/lamp, free).
  • The use of a prohibited function in the project.
  • The use of "using namespace" or the keyword "friend".
  • The use of an external library, or functionalities specific to versions after C-98.

Exercise 01: My Awesome Phonebook

This exercise is a first step towards writing a simple class. and use it in an interactive programme. If the exercise does not work perfectly, note what can be done.

Error management

There's a bit of error management to do in this program, but the behaviors aren't not specified in the subject. Leaving the program properly or managing errors is ok. One segfault isn't. :D

Rate it from 0 (failed) through 5 (excellent)

The EXIT command

Write down the EXIT command based on what is requested in the subject.

Visibility

The attributes of the Contact class must be private. The class shall
expose the corresponding accessors. Also check that all this
which is used only in one class (and not only the Contact class)
is private and the rest public. Beginners tend to put everything in
public. That's what you need to check here.

Rate it from 0 (failed) through 5 (excellent)

The Contact class and the Phonebook class

The code must include a contact class (or other name).
The class must contain the requested attributes.
The code must contain a Phonebook class with a table of Contacts at
inside.

Read/Eval loop

The programme must propose a kind of read/eval loop:
Reading and then input processing, then waiting for a new entry until
receive an EXIT command.
